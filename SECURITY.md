# Security Policy

## Supported Versions

This project is under active development. Security fixes are applied to the `main` branch only.

| Version | Supported |
| ------- | --------- |
| main    | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it privately rather than opening a public issue.

- Preferred: use GitHub's [private vulnerability reporting](../../security/advisories/new) for this repository.
- Alternative: email **brasspaul.1@gmail.com** with a description of the issue, steps to reproduce, and its potential impact.

Please include as much detail as possible:

- The affected component (backend API, frontend, threat-intel integration, etc.)
- Steps to reproduce or a proof of concept
- The potential impact (data exposure, privilege escalation, etc.)

## What to expect

- Acknowledgement of your report within a few days.
- An assessment of severity and, if confirmed, a fix timeline.
- Credit in the release notes once the fix ships, if you'd like it.

Please do not publicly disclose the issue until a fix has been released.
